sed Solution
A paperless web portal where students report campus issues with location + description photo (optional)
The system shows a live list of similar existing issues, so students can join instead of creating duplicates.
If no match exists, the student can create a new complaint, generating a single master ticket.
Each ticket is routed to the correct Office Queue (Hostel Office / Academic Office / Facilities) based on location.
Office staff assign a handler/technician, update status OPEN → IN_PROGRESS → RESOLVED, and add updates.
Students track everything on a ticket page with SLA countdown + timeline, ensuring transparency and accountability.
